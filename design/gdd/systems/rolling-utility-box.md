# 滚动功能箱系统

> **Status**: Draft
> **Author**: Codex + User
> **Last Updated**: 2026-03-21
> **Implements Pillar**: 滚动就是解谜 / 一个箱子，多种用途 / 可爱外表，硬核规则

## Overview

滚动功能箱是本项目的核心系统。玩家推动一个立方体箱子时，箱子会按格子向前翻滚，翻滚后顶面改变，顶面类型决定它此刻能做什么。这个系统同时承担了解谜、机关触发和轻战斗三个职责，是整个游戏最重要的规则来源。

它存在的目的不是增加复杂度，而是让玩家在每一步都思考“把箱子推到哪里”和“让它以什么状态到达”这两个问题。

## Player Fantasy

玩家应该感觉自己不是在搬运笨重障碍物，而是在操控一个聪明的万能工具。最理想的体验是：
- 我先看懂局面
- 我知道这次往前滚会变成什么
- 我能让同一个箱子先开门、再打敌人、最后变成钥匙

当玩家成功复用同一个箱子完成多个目标时，系统就达成了它的情感目标。

## Detailed Design

### Core Rules

1. **基础网格**
   - 游戏逻辑按方格进行。
   - 玩家每次只能向四个正交方向移动 1 格。
   - 箱子也只按四个正交方向滚动 1 格。

2. **推动前提**
   - 玩家与箱子相邻，且玩家移动方向正对箱子，才可推动。
   - 箱子目标格若被墙、关闭的门、另一个箱子占据，则推动失败。
   - 箱子目标格若有敌人，则先计算“箱子滚过去后的顶面”，再判断能否击败敌人。

3. **滚动结算**
   - 每次推动成功，箱子向目标格翻滚 90 度。
   - 翻滚不仅改变位置，也改变朝向。
   - 箱子必须完整结束翻滚动画后，才进行按钮、敌人、能源槽等交互结算。

4. **顶面功能**

| 顶面类型 | 面数 | 主要作用 | 次要说明 |
| ---- | ---- | ---- | ---- |
| 普通面 | 2 | 压普通按钮 | 最基础状态，不擅长战斗 |
| 冲击面 | 2 | 击败普通敌人 | 适合清除路径 |
| 重压面 | 1 | 压重压按钮，击败重甲敌人 | 稀有高价值状态 |
| 能源面 | 1 | 激活能源槽、启动特殊门或终端 | 偏工具型状态 |

5. **交互结算顺序**
   - 第一步：检查目标格是否为硬阻挡。
   - 第二步：若目标格有敌人，使用“翻滚后的顶面”判断是否可击败。
   - 第三步：箱子进入目标格并更新朝向。
   - 第四步：结算格内机关。
   - 第五步：刷新与该机关相连的门、平台或出口状态。

6. **机关规则**
   - 普通按钮：任意顶面都可触发。
   - 重压按钮：只有重压面触发。
   - 能源槽：只有能源面触发。
   - 默认规则下，按钮类机关要求箱子持续停留其上；箱子离开后，机关失效。
   - 某些后续关卡可引入“一次触发后保持开启”的锁存机关，但不属于 MVP 必需内容。

7. **敌人规则**
   - 玩家不直接攻击敌人。
   - 敌人默认占据 1 格，并阻挡玩家和箱子通过。
   - 普通敌人可被冲击面或重压面击败。
   - 重甲敌人只能被重压面击败。
   - 若预测顶面不满足击败条件，则推动失败，玩家和箱子都不移动。

8. **失败与重置**
   - 玩家掉入坑洞，立刻失败并允许快速重开。
   - 若关键箱子掉入坑洞或进入不可恢复状态，关卡直接判定为需重开，避免软锁处理的额外复杂度。
   - MVP 阶段优先使用“整关快速重置”，不做局部撤销系统。

### States and Transitions

| State | Entry Condition | Exit Condition | Behavior |
|-------|----------------|----------------|----------|
| 普通面朝上 | 箱子滚动后顶面为普通面 | 再次滚动或关卡重置 | 可压普通按钮，不可击败任何敌人 |
| 冲击面朝上 | 箱子滚动后顶面为冲击面 | 再次滚动或关卡重置 | 可击败普通敌人，也可压普通按钮 |
| 重压面朝上 | 箱子滚动后顶面为重压面 | 再次滚动或关卡重置 | 可压重压按钮，可击败普通敌人与重甲敌人 |
| 能源面朝上 | 箱子滚动后顶面为能源面 | 再次滚动或关卡重置 | 可激活能源槽，也可压普通按钮 |

### Interactions with Other Systems

| System | 交互方式 |
| ---- | ---- |
| 玩家移动系统 | 提供推箱输入与推动方向 |
| 敌人系统 | 提供敌人类型与脆弱面规则 |
| 机关系统 | 根据顶面类型决定按钮或能源槽是否激活 |
| 地形系统 | 坡道、传送带、旋转平台改变箱子的最终状态或位置 |
| 关卡重置系统 | 在失败或软锁前统一回到初始布局 |
| UI 系统 | 显示当前顶面图标、可触发目标提示 |
| 动画系统 | 保证滚动动画与逻辑朝向一致 |

## Formulas

本系统尽量避免复杂数值计算，核心采用确定性布尔规则。

### 按钮激活判定

```text
activated = top_face in accepted_faces
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| top_face | enum | 普通/冲击/重压/能源 | 箱子当前朝向 | 箱子滚动后朝上的功能面 |
| accepted_faces | set | 面集合 | 机关配置 | 该机关允许触发的顶面类型 |

**Expected output range**: true / false  
**Edge case**: 普通按钮的 accepted_faces 为“全部面”。

### 敌人击败判定

```text
defeated = predicted_top_face in enemy_vulnerability
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| predicted_top_face | enum | 普通/冲击/重压/能源 | 推动前预计算 | 箱子进入敌人格后会朝上的面 |
| enemy_vulnerability | set | 面集合 | 敌人配置 | 该敌人可被哪些顶面击败 |

**Expected output range**: true / false  
**Edge case**: 若结果为 false，则推动整体失败，不发生位移。

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| 箱子目标格是墙 | 推动失败，玩家不移动 | 保持规则稳定 |
| 箱子目标格是关闭门 | 推动失败 | 门是硬阻挡 |
| 箱子目标格是普通敌人，但预测顶面为普通面 | 推动失败 | 避免“错误推动也能穿过敌人” |
| 箱子目标格是普通按钮，当前顶面为能源面 | 仍可触发 | 普通按钮强调位置，不强调特殊要求 |
| 箱子从按钮上被移开 | 关联门恢复默认关闭状态 | 保持谜题清晰 |
| 箱子掉进坑里 | 关卡进入失败态并允许快速重开 | MVP 阶段优先简单稳定 |
| 玩家把敌人压在机关上 | 不支持该布局，关卡设计阶段避免 | 降低规则复杂度 |
| 两个箱子连续推动连锁 | MVP 不支持箱子推箱子 | 限制实现范围 |

## Dependencies

| System | Direction | Nature of Dependency |
|--------|-----------|---------------------|
| 网格移动 | This depends on 网格移动 | 需要稳定的格子坐标和方向输入 |
| 动画系统 | This depends on 动画系统 | 需要可控的 90 度滚动动画 |
| 机关系统 | This depends on 机关系统 | 需要按钮、门、能源槽响应 |
| 敌人系统 | This depends on 敌人系统 | 需要敌人可配置脆弱面 |
| 教学关卡系统 | 教学关卡系统 depends on this | 整个前 5 关围绕此系统展开 |
| UI 系统 | UI depends on this | 需要读取顶面状态并展示给玩家 |

## Tuning Knobs

| Parameter | Current Value | Safe Range | Effect of Increase | Effect of Decrease |
|-----------|--------------|------------|-------------------|-------------------|
| 箱子滚动时长 | 0.22s | 0.18s-0.35s | 更厚重，但节奏更慢 | 更利落，但可能不易看清 |
| 推动失败反馈停顿 | 0.08s | 0.05s-0.12s | 更明显，但更卡 | 更顺，但提示不足 |
| 顶面图标尺寸 | 中 | 小-大 | 可读性更强 | 画面更干净但信息更弱 |
| 普通敌人种类 | 1 | 1-2 | 关卡变化更多 | 更聚焦教学 |
| 重甲敌人出现节奏 | 第 4 关 | 第 4-6 关 | 更早形成压力 | 更晚但教学更平滑 |

## Visual/Audio Requirements

| Event | Visual Feedback | Audio Feedback | Priority |
|-------|----------------|---------------|----------|
| 箱子滚动成功 | 明确翻滚动画，顶面图标更新 | 轻快木质或塑料滚动声 | Critical |
| 箱子推动失败 | 轻微回弹或震动 | 短促闷音 | High |
| 普通按钮激活 | 按钮下沉，门框灯亮 | 清脆开关声 | Critical |
| 重压按钮激活 | 更明显的机械压下效果 | 更沉的机关声 | High |
| 箱子击败敌人 | 敌人弹飞或拆散成零件 | 玩具化的撞击音 | Critical |
| 能源槽激活 | 灯带点亮，终端发光 | 柔和充能音 | High |

## UI Requirements

| Information | Display Location | Update Frequency | Condition |
|-------------|-----------------|-----------------|-----------|
| 箱子当前顶面图标 | 箱子顶面 + 轻量悬浮提示 | 每次滚动后 | 始终 |
| 可激活的机关提示 | 机关上方图标或颜色边框 | 玩家接近时 | 条件式 |
| 推动失败原因 | 屏幕边缘短文本或图标 | 失败时 | 条件式 |
| 重置提示 | 屏幕角落 | 常驻 | 始终 |

## Acceptance Criteria

- [ ] 玩家可以稳定地向四个方向推动箱子 1 格
- [ ] 箱子每次滚动后，逻辑朝向与动画结果完全一致
- [ ] 普通按钮、重压按钮、能源槽都能按规则识别顶面类型
- [ ] 普通敌人与重甲敌人的击败条件完全确定，无随机因素
- [ ] 推动失败时不会产生穿模、重叠或错误位移
- [ ] 关卡重置后，所有箱子朝向与位置恢复初始状态
- [ ] 玩家仅凭画面反馈即可判断箱子当前用途
- [ ] No hardcoded values in implementation
